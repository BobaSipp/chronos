Contributing

Guidelines for contributing to Chronos.

Code of Conduct

This project is open and welcoming. All contributors are expected to treat each other with respect and professionalism.

Getting Started

Chronos is a small project with a straightforward codebase. The entire application is organized into six focused modules under lib/chronos/:

File Responsibility
version.rb Chronos::VERSION constant
ansi.rb ANSI escape code constants for terminal styling
data.rb GitData struct and collect_git_data method
cli.rb Option parsing, usage output, JSON export
tui.rb Main TUI loop, keyboard input handling
renderer.rb All rendering and layout logic

Development Workflow

  1. Fork the repository on GitHub.
  2. Create a feature branch 
    git checkout -b feat/my-feature
  3. Make your changes. Follow the existing code style:
    • Use module_function for module-level methods
    • Use # frozen_string_literal: true in all Ruby files
    • Use Unicode characters for box-drawing (not ASCII approximations)
    • Keep methods focused and single-purpose
  4. Test your changes by running Chronos in a test repository. 
    cd /tmp/test-repo
ruby /path/to/chronos/chronos.rb
  5. Submit a pull request with a clear description of what your changes do and why they're needed.

Feature Ideas

Some areas we'd love help with:

  • Commit timeline / activity over time visualization
  • Churn rate (lines added/deleted per file)
  • Author + file cross-reference ("bus factor")
  • Config file support (~/.chronosrc)
  • Gem packaging for gem install chronos
  • Unit tests for data collection and rendering

Style Guide

  • Indent with two spaces (soft tabs)
  • Prefer single-quoted strings when no interpolation is needed
  • Use &. (safe navigation) for nil checks
  • Keep lines under 100 characters where practical
  • No external dependencies — stdlib only

Questions?

Open an issue on GitHub or start a discussion. We're happy to help new contributors get started.
